# Official Rules (what actually matters for us)

Sponsor: Microsoft Corporation. Entry period: May 19 – **June 14, 2026, 11:59 PM PT**.

## Eligibility

- 18+ (parental consent if under age of majority in your residence).
- Void in Cuba, Iran, North Korea, Sudan, Syria, Crimea, Russia.
- Microsoft employees/family ineligible.

## A complete submission MUST include

1. **Working project** ("Agent") built with the required tools, meeting the challenge core requirements.
2. **Demo video, 5 minutes max**, uploaded to **YouTube or Vimeo**. Must be solely your own work (filming, editing, graphics). No third-party trademarks/copyrighted material without permission. (Microsoft trademarks/logos: limited license granted for the entry.)
3. **Project description** — features, functionality, problem solved, technologies used.
4. **Public GitHub repository** with the source code (must include a README).
5. **Architecture diagram** showing how the solution uses Microsoft Foundry / M365 Copilot / GitHub Copilot.
6. **Team member info** incl. Microsoft Learn usernames for all participants (if a team).

Submit via the **Projects** tab on the contest website.

## Entry limits and originality

- Max **3 entries per person per challenge**, 9 total. No multiple accounts.
- Entry must be **original work**, not a winner of any other contest.
- You must hold all consents/licenses for everything you submit.
- No obscene/offensive/political content etc.

## Rights you grant

Microsoft gets an irrevocable, royalty-free, worldwide license to use the entry for any purpose (including marketing). You keep ownership. Entry may be posted publicly.

## Winner selection

- One winner per challenge, selected within 21 days after entry period; notified within 7 days after judging.
- Community vote (10%) happens via Discord poll — **vote buying / automation is prohibited and voids votes**.
- Private repos created for preparation **must be made public within 90 days** (per Disclaimer).

## Pre-submission checklist (operational)

- [ ] Repo public, README present, synthetic-data disclaimer in README
- [ ] Demo video ≤ 5 min on YouTube/Vimeo, own footage only
- [ ] Architecture diagram committed (and embedded in README)
- [ ] Project description written (features / problem / tech)
- [ ] Microsoft Learn usernames for all team members at hand
- [ ] No secrets/PII anywhere in history (see compliance.md)
- [ ] At least one Microsoft IQ layer demonstrably integrated
